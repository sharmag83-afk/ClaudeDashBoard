# CLAUDE.md

Guidance for Claude Code when working in this repository.

## Project overview

A personal dashboard: a set of small, self-contained HTML apps that share a top bar (`topbar.js`) and sidebar (`sidebar.js`). Each page stores its own state in browser `localStorage` — no build step, no server, no accounts. See `README.md` for the full page list and `BUILD_DASHBOARD.md` / `SETUP.md` for background.

## Communication style

In conversation and explanations, use plain, simple language — aim for around a 7th-grade reading level, and lean on concrete analogies to make ideas land. Keep full rigor and honest feedback; simplifying the words is not the same as softening the substance.

This applies to conversational replies and explanations only. Deliverables (docs meant to be read as artifacts), formal communications (PR descriptions, commit messages, issue comments), code, and machine commands (shell commands, config) should keep their normal professional register — don't dumb these down.
